# SmartPlace Bulk — 네이버 스마트플레이스 일괄 관리

> 네이버 스마트플레이스 **전 지점 관리 올인원** 데스크톱 앱 — 사장님·프랜차이즈 본사용
> 사진·메뉴 일괄 등록 · 통계·리뷰 수집 · AI 답글 초안 · **배달앱(배민·쿠팡이츠) 손익계산서**까지 한 곳에서

> **사용 전 반드시 [면책 고지(DISCLAIMER)](DISCLAIMER.md)를 읽어주세요.**
> 이 도구는 네이버와 무관한 비공식 도구이며, 자동화는 네이버 약관과 충돌할 수 있습니다. 사용에 따른 책임은 사용자에게 있습니다. **본인 소유/관리 지점에만 사용하세요.**

## 무엇을 하나요?
수십~수백 개 지점을 **하나씩** 관리하던 일을 한 곳에서:

**네이버 스마트플레이스**
- **대표사진 일괄 등록** — 표준 이미지를 전 지점에 (대표사진으로 설정)
- **메뉴 일괄 등록** — 표준 메뉴(CSV)를 전 지점에 통일 (이름·가격·설명·사진)
- **전 지점 통계 수집** — 방문·조회·리뷰·예약수를 모아 CSV (읽기 전용)
- **전 지점 리뷰 수집** — 작성자·별점·내용을 모아 CSV (읽기 전용)
- **AI 리뷰 답글 초안** — 수집한 리뷰에 사장님 답글 초안 (Claude, *초안만*)

**배달앱**
- **배달앱 손익계산서** — 배민·쿠팡이츠·요기요 정산파일(.xlsx) → 손익계산서 엑셀 (순수 로컬 계산, 토큰비용 없음)

**고객 PC에서 고객 계정으로 실행** → 로그인·파일이 PC를 떠나지 않습니다 (Local-First).
(예외: AI 답글 초안만 본인 API 키로 리뷰 본문이 Anthropic에 전송됩니다.)

> **어떻게 시작하나요?**
> · 그냥 **쓰고 싶다** → 아래 [다운로드 & 설치](#다운로드--설치) (설치 불필요, 더블클릭)
> · **코드를 직접 돌려보거나 기여하고 싶다** → [ 소스에서 5분 실행](#소스에서-5분-실행-개발자기여자용)

## 사용 매뉴얼 (기능별)

> 컴퓨터에 익숙하지 않아도 괜찮아요. 화면에 보이는 순서대로 따라 하면 됩니다.
> 네이버 기능(사진·메뉴·통계·리뷰)은 **① 로그인 → ② 지점 불러오기**가 먼저예요.
> **배달앱 손익계산서**는 로그인 없이 바로 씁니다.

### 공통 — 시작하기
1. **네이버 로그인** — `네이버 로그인` 버튼 → 뜨는 창에서 **평소처럼** 로그인(아이디·비번·캡차·2차인증). `로그인됨 ` 이 보이면 완료. (한 번 하면 다음부턴 유지돼요)
2. **지점 불러오기** — 브랜드 번호 입력 → `지점 불러오기` → 전 지점이 목록으로 나와요. 작업할 지점을 체크하세요. (`전체 선택`/`전체 해제` 버튼도 있어요)
   > **브랜드 번호(brandSeq)** 는 네이버 스마트플레이스 *브랜드 관리* 페이지 주소의 `brandSeq=` 뒤 숫자예요.

### 대표사진 일괄 등록
1. *작업 선택*에서 **대표사진 일괄**
2. `이미지 폴더 선택` → 올릴 사진이 든 폴더 선택
3. `실행` → 선택한 지점에 한 곳씩 등록돼요 (진행률·성공/실패 표시)

> **처음엔 꼭 한 지점만** 체크해서 테스트하세요 — 실제로 네이버에 올라갑니다.

### 메뉴 일괄 등록
1. *작업 선택*에서 **메뉴 일괄**
2. `메뉴 양식(CSV) 받기` → 엑셀로 열어 메뉴를 채워요(이름·가격·설명·사진파일명·대표여부)
3. `작성한 CSV 선택` (메뉴 사진이 있으면 `메뉴 이미지 폴더`도 선택)
4. (선택) **기존 메뉴 삭제 후 통일(교체)** 체크 — 기존 메뉴가 지워지니 주의
5. `실행`

양식 예시 ([`desktop/menu_template.csv`](desktop/menu_template.csv)):
```
name,price,description,image,recommended
빠삭파전,6900,겉바속촉 빠삭파전,pajeon.jpg,Y
```

### 전 지점 통계 수집 (읽기 전용)
1. *작업 선택*에서 ** 리포트 수집** → `실행`
2. 선택 지점의 **방문·조회·리뷰·예약수**를 모아줘요
3. `CSV로 저장` → 엑셀로 열어 보세요

> 매장 정보를 **바꾸지 않는** 안전한 읽기 기능입니다. (베타: 못 읽은 지점은 `읽기 실패`로 표시 — 정직하게)

### 전 지점 리뷰 수집 (읽기 전용)
1. *작업 선택*에서 ** 리뷰 수집** → `실행`
2. **작성자·별점·내용·작성일**을 한곳에 모아 `CSV로 저장`

### AI 리뷰 답글 초안
1. 먼저 **리뷰 수집**을 하세요
2. ` AI 답글 초안` 항목을 펼쳐 **Anthropic API 키** 입력 (선택: *이 PC에 키 저장*)
3. (선택) **브랜드 지침**(말투·호칭 등) 입력 → `AI 답글 초안 생성`
4. 별점에 맞춘 답글 초안이 생성돼요 → `답글 초안 CSV 저장`

> **초안만** 만들어 줍니다. 검토·수정 후 네이버에 **직접 게시**하세요. (이 기능만 리뷰 본문이 Anthropic으로 전송됩니다)
> API 키는 [console.anthropic.com](https://console.anthropic.com) 에서 발급받아요.

### 배달앱 손익계산서 (로그인 불필요)
1. 배민·쿠팡이츠·요기요에서 받은 **정산 파일(.xlsx)** 을 준비
2. `정산 파일 선택(여러 개)` → **과세유형**(일반/간이/면세) 선택
3. 배민 정산파일이 **암호화**돼 있으면 **비밀번호** 칸에 입력
4. (선택) **식자재·인건비·임대료·공과금**을 직접 입력하면 손익계산서가 더 완성돼요
5. `손익계산서 생성` → 영업이익·당기순이익·세부 항목·경고 확인 → `손익계산서(엑셀) 저장`

> **AI/토큰비용 없이** 순수 로컬 계산이라, 파일이 PC를 떠나지 않습니다.
> 배민 정산명세서는 암호로 보호돼 있어 비밀번호가 필요해요(쿠팡·요기요는 불필요).

## 소스에서 5분 실행 (개발자·기여자용)

> 완성된 앱만 쓸 거면 위 [다운로드 & 설치](#다운로드--설치)로 충분합니다.
> 코드를 직접 돌려보거나 기여하려면 아래 3단계면 됩니다.

**전제**: Python 3.12+ 하나면 됩니다. (데스크톱 앱만 띄울 거라 Node·DB·클라우드 불필요)

**① 클론**
```bash
git clone https://github.com/yuneunmi814-cmyk/smartplace-cloud.git
cd smartplace-cloud/desktop
```

**② 실행** — 둘 중 편한 방법으로:

- **가장 쉬움**: 파일 탐색기에서 **`run.command`(Mac)** 또는 **`run.bat`(Windows)** 더블클릭 → 처음엔 자동으로 설치하고 앱이 뜹니다.
- ⌨ **터미널로**:
  ```bash
  python3 -m venv .venv && source .venv/bin/activate   # Windows: .venv\Scripts\activate
  pip install -r requirements.txt
  playwright install chromium                          # 한 번만 (가장 오래 걸리는 단계)
  python app.py
  ```

**③ 앱 창이 뜨면 성공!** 4단계 화면(로그인 → 지점 → 작업 → 실행)이 보입니다.

> 실제 등록을 해보려면 **본인 네이버 계정으로 로그인**하고 본인이 관리하는 브랜드 번호가 필요합니다.
> (로그인 정보는 PC를 떠나지 않습니다. 처음엔 **꼭 한 지점만** 테스트하세요.)

<details>
<summary>잘 안 될 때</summary>

- `python3` 없음 → [python.org](https://www.python.org/downloads/)에서 3.12+ 설치 (Windows는 설치 시 "Add to PATH" 체크)
- `playwright install` 멈춤 → 네트워크 문제. Chromium(약 150MB) 다운로드라 와이파이가 느리면 5분을 넘길 수 있어요. 다시 실행하면 이어받습니다.
- macOS에서 `run.command`가 "확인되지 않은 개발자" → 우클릭 → "열기" 한 번.
</details>

### 전체 스택(클라우드)까지 돌려보려면
백엔드(FastAPI) + 웹 콘솔(React) + 게이트웨이까지 띄우는 방법은 **[HANDOFF.md](HANDOFF.md)** 에 단계별로 있습니다.

### 저장소 구성
```
desktop/   데스크톱 앱(pywebview + Playwright) — 배포되는 실행파일 ← 위 퀵스타트 대상
gateway/   CLI 자동화 도구(일괄 사진/메뉴, 브랜드 스크랩)
backend/   FastAPI 백엔드(다계정·작업 큐·S3·라이선스/구독) + 테스트
web/       React 관리 콘솔(이미지·배포·라이선스 관리)
```
빌드: `.github/workflows/build-windows-exe.yml` (GitHub Actions에서 Windows .exe 자동 빌드)
